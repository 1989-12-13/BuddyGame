你会接到各种各样的电话，可能是求助，也可能是恶作剧。系统只会自动显示来电号码和模糊的基站定位。具体的详细地址、病情描述，都需要玩家通过与呼救者的对话来询问和确认，就像真实流程中要问“旁边有什么标志性建筑物”一样。

在一通急救电话中，接线员的动作是并行推进的，主要包括这几步：

1.  **核实关键信息**：接线员按特定流程，优先问清楚**事件发生地址、联系电话、患者基本情况和诉求目的**这“四要素”，这是决定是否派车的基础。使用**MPDS（急救医疗优先分级调度系统）**，通过标准化问题对患者情况进行分级，并为后续指导提供依据。

2.  **判断事件等级并派车**：在问询的同时，接线员会快速判断事件紧急程度，决定优先级。根据判断，遵循“就近、就急”的原则在系统内发出派车指令。救护车到达时间与派车时间和地址信息完整程度有关，进而影响患者的伤情。

3.  **派车后的跟踪与协调**：接线员或系统还需：
    *   **回拨确认**：对无声或中断的电话，按规定回拨多次确认情况。
    *   **响应催车**：接到催车电话时，需安抚来电者并告知车辆动态。
    *   **处理特殊情况**：比如告知报警人救护车特征，或帮忙协调交警开通“绿波带”保障通行。

4. **指导现场人员实施初步急救**：例如，正确指导海姆立克急救法的步骤，在游戏中可以设计为选择正确的操作顺序和动作要点。


游戏简要流程：

| 环节 | 时间要求 | 说明 |
| :--- | :--- | :--- |
| **发出派车指令** | **1分钟内**、**27秒**（平均） | 这是硬性规定，必须在问询同时或极短时间内完成。 |
| **受理（问询+派单）总时长** | **43.14秒**（平均）、**1-2分钟**（熟练后） | 从接起到完成核心操作的全程时间。 |
| **急救站出车** | **白天2分钟/晚上3分钟内**、**3分钟内**、**1.03分钟**（平均） | 这是对急救站的要求，指从收到指令到车辆驶出。 |
| **电话急救指导** | **4分钟内**（经典案例） | 在等车的“黄金空档”里进行，不影响派车，甚至能挽救生命。 |

游戏设计：

1.  **时间压力是核心玩法**：可以把“1分钟内派车”、“43秒完成受理”等硬指标，设计成游戏中的**核心评分或过关条件**，决策速度也会影响事件结果。正确、快速的响应会获得声誉和“奖章”，而错误（比如派错单位、给出错误急救指导）会导致负面评价。可以通过上帝视角或报警人的言语来反馈“伤员的健康状况随时间的变化”，模拟真实的紧张感。
2.  **信息收集是第一步挑战**：将现实中的“四要素”问询流程，设计成游戏的核心交互。玩家的目标就是在慌乱的通话中，快速、准确地抓取关键信息。
3.  **“并行处理”的成就感**：可以设计一种**“双线程”操作机制**，让玩家在问询的同时，需要在电脑上登记事件。这种并行处理能极大提升游戏的沉浸感和紧张度。
4.  **“黄金4分钟”的抉择**：派车后的等待时间，设计成**提供正确急救指导的关键决策窗口**，作为自然的科普教学场景。
